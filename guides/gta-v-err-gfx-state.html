How to Fix ERR_GFX_STATE Error in GTA V — 2026 Guide

⚡ Quick Fix

Switch GTA V from DirectX 11 to DirectX 10 in Settings → Graphics. This single change fixes ERR_GFX_STATE for the majority of players.

What Is ERR_GFX_STATE?

ERR_GFX_STATE is a graphics pipeline crash in GTA V. It occurs when the game's renderer encounters a state it can't recover from — often caused by DirectX conflicts, MSAA settings, driver issues, or overlay software interfering with rendering.

Step-by-Step Fix

1

Switch DirectX Version

In GTA V: Settings → Graphics → DirectX Version — switch from DX11 to DX10 (or vice versa). Also try switching between Vulkan and DX11 if available. Restart the game after changing.

2

Disable MSAA

MSAA (Multisample Anti-Aliasing) is a common trigger for this error. Go to Settings → Graphics → MSAA and set it to Off. You can use FXAA instead if you still want anti-aliasing.

3

Update GPU Drivers

Download and install the latest GPU drivers. Use the clean install option. If you're on a very new driver that introduced the issue, try rolling back to a previous stable version.

4

Disable Overlays

Discord, Steam, NVIDIA GeForce Experience, and Xbox Game Bar overlays can conflict with GTA V's renderer:

  • Discord: Settings → Game Overlay → Disable
  • Steam: Right-click GTA V → Properties → Disable Steam Overlay
  • GeForce Experience: Settings → Disable In-Game Overlay
  • Windows: Settings → Gaming → Disable Xbox Game Bar
5

Delete GTA V Settings File

Corrupted graphics settings can cause persistent crashes. Delete the settings file to reset to defaults:

C:\Users\[YourName]\Documents\Rockstar Games\GTA V\settings.xml

The game will recreate this file with default settings on next launch.

💎 Pro Tip

GTA V is notoriously heavy on VRAM. If you have a 4GB VRAM GPU, keep Texture Quality at Normal and avoid MSAA entirely. For 6GB+ GPUs, you can safely use higher textures, but keep an eye on the VRAM usage bar in the Graphics settings. A modern 8GB VRAM GPU eliminates most GTA V graphics issues entirely.
